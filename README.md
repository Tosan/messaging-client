# messaging-client

## Architecture

This is a **multi-module Spring Boot starter** (Java 17) providing messaging service abstractions for Iranian SMS providers. Parent POM is `pom`-packaged; child modules produce `jar` artifacts.

### Modules

| Module | Purpose |
|---|---|
| `messaging-client-spring-boot-starter` | Core starter — `MessagingService` interface + provider implementations (KaveNegar, ArmaghanNegar) |
| `chapar-messaging-client-starter` | Separate auto-configuration for the Chapar messaging provider |
| `messaging-client-sample` | Non-deployable sample app showing usage |

### Provider selection (core starter)

The `MessagingService` interface exposes four operations: `sendOneToManyMessage`, `sendManyToManyMessage`, `sendOneToOneMessage`, `sendOtpMessage`. The active implementation is selected via the `messaging-client.provider` property:

- **`KAVE_NEGAR`** — `KaveNegarMessagingService` uses `RestTemplate` with URL-encoded forms. Configuration class: `KaveNegarClientConfiguration`.
- **`ARMAGHAN_NEGAR`** — `ArmaghanNegarMessagingService` uses a **Feign client** generated by OpenAPI Generator from `src/main/resources/openapi.json`. Configuration class: `ArmaghanNegarClientConfiguration`. The generated Feign interface is `WebserviceApi`.

Both are activated by `@ConditionalOnProperty` and produce a `"messaging-service-client"` bean. The `"messaging-client-clientConfig"` bean holds shared HTTP properties under the `messaging-client` prefix.

### Chapar module (separate starter)

`chapar-messaging-client-starter` is independent — it does **not** implement `MessagingService`. Its interface is `ChaparMessagingService` with a single `sendEvent` method. Uses `RestTemplate` with **OAuth2 client_credentials** token management (token cached in Redis via Tedisson, auto-refreshed on 401). Configuration prefix: `messaging-client.chapar`.

### Property model

`MessagingClientConfig` extends `HttpClientProperties` (from the internal `tosan-httpclient` library) and adds provider selection plus nested config for each provider. `ChaparClientProperties` likewise extends `HttpClientProperties` and adds `clientId`/`clientSecret` for OAuth2.

### Key internal dependencies

- **`tosan-httpclient-spring-boot-starter`** — shared HTTP client infrastructure (`HttpClientProperties`, `AbstractFeignConfiguration`, `AbstractHttpClientConfiguration`)
- **`tosan-httpclient-spring-boot-resttemplate-starter`** — `ExternalServiceInvoker` helper for RestTemplate-based clients
- **`tedisson-spring-boot-starter`** — Redis caching (used by Chapar for token caching)
- **`tosan-mask-starter`** — sensitive data masking in logs

### Test convention

Only files matching `**/*UTest.java` are executed by Surefire. There are currently no test files in the project.

### Spring auto-configuration entries

- Core starter (`META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports`): `MessagingClientConfiguration`, `ArmaghanNegarClientConfiguration`, `KaveNegarClientConfiguration`
- Chapar starter: `ChaparPropertiesAutoConfiguration`, `ChaparClientAutoConfiguration`

## Build / Test Commands

```bash
# Build everything (no tests)
mvn -B package --file pom.xml

# Build with tests (only *UTest.java files are picked up by Surefire)
mvn -B test

# Build a single module
mvn -B package -pl messaging-client-spring-boot-starter

# Run a single test class
mvn -B test -pl messaging-client-spring-boot-starter -Dtest=ClassNameUTest

# Regenerate the ArmaghanNegar Feign client from OpenAPI spec
mvn -B process-sources -pl messaging-client-spring-boot-starter -P generateClient
```
